# ABIZ POS Desktop Offline Plan

This project now has a desktop/offline foundation in the `desktop/` folder.

## What Runs Where

- Railway: central FastAPI backend, PostgreSQL database, owner web dashboard, reports.
- Client Windows PC: Electron desktop POS app with local SQLite storage.

## Offline Flow

1. Shopkeeper opens desktop app.
2. Products are synced from the online backend when internet is available.
3. USB QR/barcode scanner enters the product code into the scanner input.
4. Sale is saved locally in SQLite, even without internet.
5. Invoice can be printed immediately.
6. When internet returns, pending sales sync to the Railway backend.
7. Backend updates sales records, product stock, and Digi Khata reports.

## Desktop Setup

```bash
cd desktop
npm install
npm start
```

Build Windows installer:

```bash
npm run build
```

## Desktop Settings

In the desktop app Settings screen:

- Backend URL: `https://your-railway-domain.up.railway.app/api/v1`
- JWT Token: token generated by logging into the backend.

Later this should be replaced with a full desktop login screen.

## Sync API

The desktop app sends pending sales to:

```text
POST /api/v1/sales/sync/offline
```

Payload:

```json
{
  "sales": [
    {
      "client_sale_id": "DESK-1780000000000",
      "created_at": "2026-06-16T12:00:00Z",
      "items": [
        {
          "product_id": 1,
          "quantity": 2,
          "unit_price": 500
        }
      ]
    }
  ]
}
```
